| **Step**                                                           | **Action / Command**                                                                    | **Purpose / Notes**                                                                                                                                                                                                |
| ------------------------------------------------------------------ | --------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 🧩 **1. Check CloudShell storage before deployment**               | `df -h`                                                                                 | AWS CloudShell persistent storage (`/home/cloudshell-user`) is limited to approximately **1 GB**. Example: `/dev/loop0 974M 146M 761M 17% /home/cloudshell-user`. Verify available space before running Terraform. |
| 🧹 **2. Create temporary Terraform cache (pre-deployment)**        | `mkdir -p /tmp/terraform-cache`                                                         | Creates a temporary cache location outside the limited CloudShell user storage.                                                                                                                                    |
| 🔗 **3. Create .terraform symlink (if it does not already exist)** | `ln -s /tmp/terraform-cache .terraform`                                                 | Redirects Terraform's `.terraform/` directory to `/tmp/terraform-cache` to avoid consuming CloudShell persistent storage.                                                                                          |
| ✅ **4. Verify symlink**                                            | `ls -la`                                                                                | Confirm the symlink is active. Expected output: `.terraform -> /tmp/terraform-cache`                                                                                                                               |
| 📦 **5. Deploy Terraform**                                         | `cd /home/cloudshell-user/lacework/aws-xxxx`<br>`terraform init`<br>`terraform apply`   | Run Terraform inside the project directory. Provider downloads and module cache will be stored through the symlink.                                                                                                |
| 📊 **6. Check Terraform cache size**                               | `du -sh .terraform`                                                                     | Verify how much space Terraform providers and modules are consuming.                                                                                                                                               |
| 🗑️ **7. Safe to delete .terraform cache**                         | `rm -rf .terraform`                                                                     | The `.terraform/` directory contains downloaded providers and modules only. It can be safely deleted and recreated when needed.                                                                                    |
| 🔄 **8. Recreate .terraform after deletion**                       | `terraform init`                                                                        | Terraform automatically recreates `.terraform/` and re-downloads required providers and modules.                                                                                                                   |
| 🔁 **9. Reuse in same session**                                    | `ls -la`                                                                                | Confirms `.terraform -> /tmp/terraform-cache` symlink is active and usable during the session.                                                                                                                     |
| ⚠️ **10. Session restart behavior**                                | *(No command)*                                                                          | `/tmp` is ephemeral and may be cleared when CloudShell restarts. The symlink may remain, but the target directory may no longer exist.                                                                             |
| 🔄 **11. Recreate cache after restart (if required)**              | `mkdir -p /tmp/terraform-cache`                                                         | Recreates the temporary cache location. If the symlink already exists, no need to recreate it.                                                                                                                     |
| 🧹 **12. Optional cleanup after deploy**                           | `rm -rf .terraform`                                                                     | Safe to remove Terraform cache after deployment to reclaim storage. You can reclaim it from Terraform init                                                                                                                                               |
| 📁 **13. Keep only essential files**                               | Keep: `main.tf`, `terraform.tfstate`, `terraform.tfstate.backup`, `.terraform.lock.hcl` | These files are lightweight and required for future `terraform plan`, `terraform apply`, `terraform destroy`, or infrastructure updates.                                                                           |

Note: Never delete terraform.tfstate if you may need to run terraform destroy later. Only .terraform/ is considered disposable cache.

```bash
$ cd /home/cloudshell-user/lacework/aws-997159613128

$ du -sh .[!.]* * 2>/dev/null | sort -hr
742M    .terraform

60K     tfplan.json
8.0K    .terraform.lock.hcl
4.0K    main.tf

$ mkdir -p /tmp/terraform-cache
$ ln -s /tmp/terraform-cache .terraform

$ ls -la
lrwxrwxrwx. 1 cloudshell-user cloudshell-user 20 Jun 10 11:55 .terraform -> /tmp/terraform-cache
````
