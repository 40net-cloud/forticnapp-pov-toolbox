## FortiCNAPP Image Scanner Methods Comparison


| Method                              | Primary Use                                                                   | Where Scanning Runs                                         | How It Is Triggered                                                                                          | Required Components                                                                         | Configuration Method                                                                               | How It Works                                                                                                                                                                                                                                                                                                                                                                                                 |
| ----------------------------------- | ----------------------------------------------------------------------------- | ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Local Scanner (Inline Scanner)**  | Developer or manual container image assessment                                | Developer laptop / local VM                                 | CLI command (on-demand)  **lw-scanner evaluate ubuntu:latest**                                                                                  | Docker must be installed and running to pull and analyze container images                   | Inline scanner **executable binary (`lw-scanner`)** configured using `./lw-scanner configure auth` | `lw-scanner` pulls the image (if not local), extracts packages and libraries, sends metadata to FortiCNAPP for vulnerability assessment, and displays results locally. Optionally saves results to FortiCNAPP using `--save`.                                                                                                                                                                                |
| **CI Pipeline (Inline Scanner)**    | Security scanning during the build process before pushing images              | CI/CD runner or build agent (Jenkins, GitHub Actions, etc.) | **Pipeline step / CI job**                                                                                   | Docker runtime available in CI environment to build and analyze images                      | Inline scanner **binary integrated into CI pipeline scripts**                                      | Uses the same inline scanner engine as local scanning but runs automatically during the image build stage. The pipeline can fail using exit codes if security policies are violated before pushing the image to a registry.                                                                                                                                                                                  |
| **Platform Scanner**                | Continuous scanning of **public or internet-accessible registries**           | FortiCNAPP cloud platform - API                                   | **Automated registry integration** (Auto polling, registry notifications, on-demand scans)                   | Registry integration configured in FortiCNAPP (no local runtime required)                   | Configured through the **FortiCNAPP Console**                                                      | FortiCNAPP connects directly to public container registries, automatically pulls images, scans them for vulnerabilities, and displays results in the FortiCNAPP UI.                                                                                                                                                                                                                                          |
| **Proxy Scanner**                   | Scanning **private/on-prem registries without exposing them to the internet** | Customer environment **(VM or Kubernetes cluster)**         | **Continuous service + API / registry events** (Auto polling, Registry notification, On-demand API via curl) | **Docker runtime (VM deployment)** OR **Kubernetes container runtime (containerd / CRI-O)** | Configuration file **`config.yml`** defining registry connections and FortiCNAPP integration token | Proxy scanner runs as a **containerized service** inside the private network. When deployed on a VM it runs as a **Docker container**, and when deployed in Kubernetes it runs as a **Pod managed by Kubernetes using containerd or CRI-O**. The scanner pulls images locally from private registries, extracts metadata and package information, and sends vulnerability assessment requests to FortiCNAPP. |
| **Kubernetes Admission Controller** | Enforce container security **before deployment in Kubernetes clusters**       | Kubernetes cluster                                          | **Admission webhook triggered during deployment**                                                            | Kubernetes cluster, Admission Controller webhook, and Proxy Scanner for image access        | Installed using **Helm chart** with webhook certificates                                           | Intercepts Kubernetes deployment requests and evaluates container images against FortiCNAPP security policies before allowing the workload to be deployed.
